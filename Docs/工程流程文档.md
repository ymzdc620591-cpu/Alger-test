# 壳木AI · Unity 3D 场景游戏工程流程文档

> 策略：**基础设施层固定，玩法层留空**
> 每次收到新需求，按本文档的阶段顺序逐步实现，不跨步。

---

## 一、目录结构规范

```
壳木AI/
└── Assets/
    ├── Starter/                  ← 基础设施层（稳定，轻易不改）
    │   ├── Core/                 # 核心系统：事件总线、单例基类、服务定位器
    │   ├── Bootstrap/            # 启动流程：场景加载器、初始化序列
    │   ├── Runtime/              # 运行时工具：对象池、计时器、协程管理
    │   ├── Authoring/            # 编辑器扩展：自定义Inspector、Gizmos辅助
    │   ├── UI/                   # UI基础框架：面板管理、弹窗系统
    │   ├── Debug/                # 调试工具：日志、性能监控、作弊码
    │   ├── Scenes/               # 基础场景：Bootstrap场景、Loading场景
    │   ├── Resources/            # 全局资源：字体、公共材质、图标
    │   └── Gizmos/               # 编辑器图标资源
    │
    └── Game/                     ← 玩法层（每个需求在此扩展）
        ├── Scripts/              # 游戏逻辑脚本
        │   ├── Player/           # 玩家相关
        │   ├── Enemy/            # 敌人/NPC
        │   ├── World/            # 世界/场景逻辑
        │   ├── System/           # 游戏系统（战斗、任务、背包…）
        │   └── Data/             # ScriptableObject 数据定义
        ├── Prefabs/              # 预制体
        │   ├── Player/
        │   ├── Enemy/
        │   ├── Environment/
        │   └── VFX/
        ├── Scenes/               # 游戏场景文件
        ├── UI/                   # 游戏UI（依赖Starter/UI框架）
        ├── Audio/                # 音效与音乐
        ├── Art/                  # 美术资源（模型/贴图/动画）
        │   ├── Models/
        │   ├── Textures/
        │   ├── Materials/
        │   └── Animations/
        └── Config/               # 游戏配置（ScriptableObject实例）
```

## 二、分层架构说明

| 层级 | 目录 | 职责 | 可依赖 |
|------|------|------|--------|
| 基础设施层 | `Starter/` | 与游戏无关的通用框架 | 仅Unity API |
| 玩法层 | `Game/` | 具体游戏逻辑 | 可依赖 Starter |
| 数据层 | `Game/Data/` `Game/Config/` | ScriptableObject配置 | 不依赖逻辑脚本 |

**规则：** `Starter/` 的代码不得引用 `Game/` 的任何类型。

## 三、功能开发标准流程

收到一个新需求时，按以下 **6个步骤** 顺序执行：

### Step 1 · 需求拆解
```
□ 用一句话描述这个功能的核心行为
□ 列出涉及的游戏对象（角色/场景物件/UI）
□ 列出输入（触发条件）和输出（结果/反馈）
□ 确认是否需要持久化（存档）
```

### Step 2 · 数据定义（Data/Config）
```
□ 为功能创建 ScriptableObject 数据类（如有配置需求）
   路径：Assets/Game/Data/Xxx定义.cs
   实例：Assets/Game/Config/Xxx配置.asset
□ 在数据类中只放纯数据字段，不写逻辑
```

### Step 3 · 目录与脚本骨架
```
□ 在 Game/Scripts/ 对应子目录下新建脚本文件
□ 先写类的骨架：字段声明 + 空方法签名 + 注释说明
□ 在 Game/Prefabs/ 下准备对应预制体（可先用空GameObject占位）
```

### Step 4 · 核心逻辑实现
```
□ 实现 Step 3 中各空方法的具体逻辑
□ 事件通信优先走 Starter/Core/ 的事件总线，不直接跨模块GetComponent
□ 需要对象复用的走 Starter/Runtime/ 的对象池
□ 每个方法控制在 20 行以内；超过则拆分
```

### Step 5 · 场景集成与测试
```
□ 在对应场景中挂载脚本/预制体
□ 在 Unity Editor 中 Play Mode 验证主流程
□ 验证边界情况（快速连点、极值输入、场景切换后状态）
□ 检查 Console 无报错、无警告
```

### Step 6 · 收尾
```
□ 清理调试用的 Debug.Log
□ 为公开字段填写 Tooltip
□ git add + git commit（提交信息格式见第五节）
```

## 四、3D 场景开发专项规范

### 4.1 场景结构（Hierarchy 层级）
```
[Scene Root]
├── --Environment--
│   ├── Terrain
│   ├── StaticMeshes
│   └── Lighting
├── --Gameplay--
│   ├── Player
│   ├── Enemies
│   └── Interactables
├── --UI--
│   └── Canvas (Screen Space - Overlay)
├── --Managers--          ← 不可见的管理器对象
│   ├── GameManager
│   ├── AudioManager
│   └── CameraManager
└── --Debug--             ← 仅编辑器可见
```

### 4.2 相机系统
```
□ 使用 Cinemachine Virtual Camera 管理镜头
□ 镜头切换通过优先级(Priority)控制，不手动开关 Camera
□ 所有镜头预制体放在 Game/Prefabs/Camera/
```

### 4.3 物理与碰撞规范
| Layer | 用途 |
|-------|------|
| Default | 静态环境 |
| Player | 玩家 |
| Enemy | 敌人 |
| Interactable | 可交互物体 |
| Projectile | 子弹/投射物 |
| UI | UI遮挡检测 |

### 4.4 性能基准
```
□ 同屏 Draw Call < 200（移动端 < 100）
□ 静态物体勾选 Static → Batching
□ 距离 > 50 单位的物体启用 LOD
□ 粒子系统最大粒子数上限 500
```

## 八、UI 系统规范

> 本工程使用栈式 UI 管理框架（参考 CodeY/PanelManager 模式）。
> **所有 UI 代码必须遵循本节规范**，不得绕过 UIManager 直接操作 Canvas。

### 8.1 核心类说明

| 类 | 位置 | 职责 |
|----|------|------|
| `UIManager` | `Starter/UI/` | 非 MonoBehaviour 单例，栈式管理所有面板 |
| `UIPanel` | `Starter/UI/` | 所有面板 prefab 根节点必须挂载的基类 |
| `ResManager` | `Starter/Runtime/` | 统一资源加载入口，封装 `Resources.Load` |
| `ExtStack<T>` | `Starter/Core/` | 支持按引用移除和按深度索引的自定义栈 |

### 8.2 面板 Prefab 制作规则

```
□ prefab 根节点必须挂载 UIPanel（或其子类），且仅挂一次（DisallowMultipleComponent）
□ prefab 存放路径：Assets/Resources/UI/XxxPanel.prefab
   → 路径即为加载 key，例如 "UI/HUDPanel"
□ 不要在 prefab 上添加 Canvas 组件（Canvas 由 UIManager 自动创建包裹层管理）
□ RectTransform 锚点/偏移在 prefab 中设好，UIManager 会原样还原
```

### 8.3 面板层级结构（运行时自动生成）

```
[UIRoot]                        ← DontDestroyOnLoad，UIManager.Init() 创建
  └── XxxPanelCanvas            ← 每个面板独享一个 Canvas（自动 SortOrder）
        └── Content             ← RectTransform 全拉伸
              └── XxxPanel      ← prefab 实例，UIPanel 组件在此
```

> `UIPanel.FadeOut()` 默认销毁 `transform.parent.parent`（即 XxxPanelCanvas）。
> 自定义退出动画时重写 `FadeOut()`，动画结束后再调 `base.FadeOut()`。

### 8.4 PanelAttr 配置（Inspector 中设置）

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `autoScale` | true | 入场时播放缩放动画（需 CanvasPanel prefab 带 Animator） |
| `showBlur` | true | 在面板下方显示半透明黑色遮罩 |
| `forceClose` | false | 点击遮罩自动调用 PopPanel 关闭 |

### 8.5 面板代码模板

```csharp
// 路径：Assets/Game/UI/XxxPanel.cs
using UnityEngine;
using Starter.UI;

namespace Game.UI
{
    public class XxxPanel : UIPanel
    {
        // 在此声明面板内的 UI 引用（Button、Text 等）

        protected override void OnShow()    // 可选：面板显示时初始化数据
        {
        }

        protected override void OnHide()    // 可选：面板隐藏时清理
        {
        }

        public override void OnPop()        // 可选：被 Pop 前保存数据
        {
        }

        public override void FadeOut()      // 可选：自定义退出动画
        {
            // 动画播放完毕后调：
            base.FadeOut();
        }
    }
}
```

### 8.6 打开 / 关闭面板

```csharp
// 打开（推入栈顶）
UIManager.Inst.PushPanel("UI/HUDPanel");

// 打开并拿到子类引用
var hud = UIManager.Inst.PushPanel<HUDPanel>("UI/HUDPanel");

// 关闭栈顶面板
UIManager.Inst.PopPanel();

// 关闭指定面板（栈内任意位置）
UIManager.Inst.PopPanel(panel);

// 关闭所有面板
UIManager.Inst.PopAllPanels();

// 查询栈内是否已存在某面板（防重复弹出）
bool exists = UIManager.Inst.CheckHavePanel("HUDPanel");
```

### 8.7 资源加载规范（ResManager）

```csharp
// 加载 prefab 引用（不实例化）
var prefab = ResManager.LoadGameObjectSync("UI/HUDPanel");

// 加载并实例化
var go = ResManager.InstantiateGameObjectSync("UI/HUDPanel");

// 加载任意类型资源
var clip = ResManager.LoadAssetSync<AudioClip>("Audio/BGM_Main");
```

> 所有游戏内资源加载统一走 `ResManager`，不直接调用 `Resources.Load`。

### 8.8 命名规范（UI 专项）

| 类型 | 规范 | 示例 |
|------|------|------|
| 面板脚本 | `XxxPanel` | `HUDPanel`, `PausePanel` |
| 面板 prefab | `XxxPanel.prefab` | `HUDPanel.prefab` |
| Resources 路径 | `UI/XxxPanel` | `"UI/HUDPanel"` |
| 面板内 UI 字段 | `_xxxText` / `_xxxBtn` | `_scoreText`, `_closeBtn` |

### 8.9 禁止事项

```
✗ 不得在场景 Hierarchy 中手动摆放带 Canvas 的 UI 面板（除 HUD 等常驻界面外）
✗ 不得直接 SetActive 控制面板显隐，统一走 PushPanel / PopPanel
✗ 不得跨面板直接 GetComponent，通过 EventBus 传递数据
✗ 不得在 Starter/UI/ 内引用 Game/ 层的任何类型
```
